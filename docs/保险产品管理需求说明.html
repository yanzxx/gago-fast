保险产品管理需求说明（精简版）

1. 建设目标

  • 统一管理保险产品信息，支持新增、查看、编辑、上架、下架。
  • 按农场维度控制数据可见范围，管理员可查看全部农场。
  • 为后续“投保管理”模块提供标准化产品数据来源。

2. 范围边界

  • 本期仅建设“保险产品管理”页面与接口。
  • 不包含移动端。
  • 不包含复杂审核流（如需“审核后上架”，建议二期补充）。

3. 角色与权限

  • 农场用户：仅可查看和维护本农场保险产品。
  • 管理员：可查看和维护所有农场保险产品。

4. 页面与交互

  • 页面结构：上方筛选区 + 下方列表区。
  • 筛选条件：养殖场、产品名称、产品状态。
  • 列表操作：详情、编辑、上架/下架、删除。
  • 顶部按钮：新增产品。

5. 数据字段设计

字段说明是否必填
farmId养殖场ID是（管理员可选，农场用户自动带入）
productName保险产品名称是
productCode保险产品编码（唯一）是
insuranceType险种类型（字典）是
coverageScope保障范围说明是
premiumRate费率（%）是
maxCompensation最高赔付金额是
status状态：DRAFT/ON_SHELF/OFF_SHELF系统维护
remark备注否

6. 业务规则

  • 产品编码全局唯一。
  • 已上架产品允许下架，不允许直接删除（需先下架）。
  • 删除采用逻辑删除（delete_flag）。
  • 费率、赔付金额必须为大于等于0的数值。

7. 接口清单（建议）

  • GET /biz/insProduct/page：分页查询
  • GET /biz/insProduct/detail：详情
  • POST /biz/insProduct/add：新增
  • POST /biz/insProduct/edit：编辑
  • POST /biz/insProduct/onShelf：上架
  • POST /biz/insProduct/offShelf：下架
  • POST /biz/insProduct/delete：删除

8. 验收标准（最小）

  • 可完成新增、查询、编辑、上/下架、删除全流程。
  • 权限隔离生效：农场用户无法查看其他农场产品。
  • 列表分页、筛选、状态展示正常。
