理赔管理需求说明（精简版）

1. 建设目标

  • 统一管理理赔申请、材料、处理进度与结果。
  • 结合投保资料与保后监控数据，提升理赔判定效率。
  • 沉淀理赔台账与报告，支持导出存档。

2. 范围边界

  • 本期仅做 Web 端，不包含移动端。
  • 本期不做“审核动作”功能，仅展示审核流程节点与状态。
  • 不建设外部支付、自动赔付、第三方司法鉴定对接。

3. 角色与权限

  • 农场用户：仅查看本农场理赔记录，提交补充材料。
  • 管理员：查看全部农场理赔记录，维护理赔状态与结果。
  • 权限按农场隔离，超级管理员可跨农场查看。

4. 页面结构

  • 筛选区：养殖场、理赔单号、保单号、理赔状态、申请时间范围。
  • 列表区：理赔记录分页列表。
  • 详情弹窗：基本信息、损失信息、材料清单、流程节点、处理结果。
  • 操作弹窗：新增理赔、补充材料、处理结果登记。

5. 核心字段

字段说明是否必填
farmId养殖场ID是
claimNo理赔单号是（唯一）
policyNo保单号是
applyNo投保单号否
insuredName被保主体是
claimType理赔类型（死亡/疾病/意外等）是
lossCount损失数量是
claimAmount申请理赔金额是
occurTime出险时间是
status状态（待受理/处理中/已结案/已拒赔）系统维护
evidenceFiles证明材料（图片/附件）否
flowNode流程节点（仅展示）系统维护
resultRemark处理结果说明结案时必填

6. 功能清单

  • 理赔记录分页查询（支持多条件筛选）。
  • 新增理赔申请（含附件上传）。
  • 查看理赔详情（资料 + 监控摘要 + 流程节点）。
  • 补充材料上传。
  • 理赔处理结果登记（通过/拒赔/结案）。
  • 理赔报告导出（PDF/Word 其一，建议先 Word）。

7. 业务规则

  • 同一保单可发起多笔理赔，但理赔单号必须唯一。
  • 理赔状态流转：待受理 -> 处理中 -> 已结案/已拒赔。
  • 已结案或已拒赔记录不可删除，仅允许查看和导出。
  • 处理结果变更需记录操作人、时间、说明。

8. 接口建议

  • GET /biz/claimManage/page：理赔分页查询
  • GET /biz/claimManage/detail：理赔详情
  • POST /biz/claimManage/add：新增理赔
  • POST /biz/claimManage/supplement：补充材料
  • POST /biz/claimManage/handle：处理结果登记
  • GET /biz/claimManage/exportReport：导出理赔报告

9. 验收标准

  • 可完成“查询-新增-详情-补充材料-结果登记-导出报告”闭环。
  • 权限隔离正确：农场用户只能查看本农场数据。
  • 流程节点可展示，处理过程有留痕。
