移动端-首页模块需求说明

文档类型：移动端某模块需求文档

模块名称：首页模块

版本：V1.0

日期：2026-03-05

1. 模块目标

  1. 在首页实时可视化展示核心养殖数据，帮助用户快速掌握经营状态。
  2. 突出设备/项圈异常，降低异常发现与处理延迟。
  3. 集中展示发情提醒、配种提醒、非法离栏等待办，提高处理效率。

2. 功能范围

  1. 核心指标展示：总存栏量、在栏数等。
  2. 异常展示：设备异常、项圈异常。
  3. 数据穿透：点击指标/异常项可进入对应详情页。
  4. 待办展示：发情提醒、配种提醒、非法离栏。

3. 页面结构

  1. 顶部信息区：当前养殖场/组织、最后更新时间。
  2. 核心指标区：以卡片方式展示关键数字。
  3. 异常概览区：按类型展示异常数量与状态标识。
  4. 待办事项区：展示待处理事项数量与摘要列表。

4. 详细需求

功能项 需求描述 交互要求
核心指标展示 展示总存栏量、在栏数等核心养殖数据，默认展示当前用户权限范围内数据。 点击指标卡可跳转详情页；支持下拉刷新数据。
设备/项圈异常 展示设备异常数、项圈异常数，按未恢复/未处理口径统计。 异常项高亮显示；点击进入异常列表并自动带筛选条件。
数据详情穿透 支持从首页卡片/列表穿透至详情页，查看明细记录。 需保留来源筛选条件，便于返回后状态不丢失。
待办事项聚合 集中展示发情提醒、配种提醒、非法离栏等待办事项。 显示每类待办数量与前N条摘要；点击进入对应处理页面。

5. 业务规则

  1. 数据权限：仅展示当前登录用户可见组织/养殖场数据。
  2. 统计口径：异常与待办均按“待处理”状态统计。
  3. 刷新机制：支持手动刷新，展示“最后更新时间”。
  4. 空态与错误：无数据时展示空态文案；接口异常时提供重试。

6. 非功能要求

  1. 性能：首页首屏关键信息加载目标 ≤ 2 秒（常规网络）。
  2. 稳定性：单个分区接口失败不影响整页渲染（分区降级）。
  3. 安全性：接口必须登录鉴权并执行数据权限校验。
  4. 兼容性：适配主流安卓/iOS手机分辨率。

7. 验收标准

  1. 首页可展示“总存栏量、在栏数、设备异常、项圈异常”。
  2. 支持点击穿透到详情页并携带筛选条件。
  3. 发情提醒、配种提醒、非法离栏可集中展示并跳转处理。
  4. 支持刷新、空态、错误态展示。
  5. 不同权限用户看到的数据范围正确。
