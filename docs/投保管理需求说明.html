投保管理需求说明（精简版）

1. 建设目标

  • 支持多条件筛选投保记录，快速定位目标保单。
  • 支持查看保单详情，形成“产品-投保-保单”数据闭环。
  • 支持按农场维度进行数据隔离与权限控制。

2. 范围边界

  • 本期仅做 Web 端，不包含移动端。
  • 本期不做审核流功能，仅做登记、查询、详情查看。
  • 理赔、保后监控、整改等能力不在本期范围。

3. 角色与权限

  • 农场用户：仅查看和维护本农场投保记录。
  • 管理员：查看全部农场投保记录。

4. 页面结构

  • 上部：筛选区（养殖场、投保单号、保单号、投保状态、产品名称、时间范围）。
  • 下部：投保记录列表（分页）。
  • 弹窗：新增投保、投保详情。

5. 核心字段

字段说明是否必填
farmId养殖场ID是
applyNo投保单号是（唯一）
policyNo保单号否（出单后回填）
productCode保险产品编码是
insuredName被保人/主体名称是
speciesType投保畜种是
insuredCount投保数量是
insuredAmount投保金额是
premiumAmount保费金额是
status状态（待提交/已提交/已生效/已失效）系统维护
startDate/endDate保险期间是
remark备注否

6. 功能清单

  • 投保记录分页查询。
  • 按条件筛选：养殖场、单号、状态、产品、时间范围。
  • 新增投保记录。
  • 查看保单详情。
  • 作废/删除投保记录（逻辑删除）。

7. 业务规则

  • 投保单号唯一，不允许重复。
  • 仅允许对未生效保单进行编辑或删除。
  • 开始日期不得晚于结束日期。
  • 投保数量、投保金额、保费金额必须大于0。

8. 接口建议

  • GET /biz/insApply/page：分页查询投保记录
  • GET /biz/insApply/detail：投保详情
  • POST /biz/insApply/add：新增投保记录
  • POST /biz/insApply/edit：编辑投保记录
  • POST /biz/insApply/delete：删除投保记录

9. 验收标准

  • 可完成查询、新增、详情、删除完整闭环。
  • 权限隔离正确：农场用户不可查看其他农场记录。
  • 筛选与分页准确，状态展示清晰。
