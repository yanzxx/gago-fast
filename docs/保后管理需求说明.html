保后管理需求说明（精简版）

1. 建设目标

  • 基于项圈数据持续监控牲畜存活与位置状态。
  • 识别并预警异常场景（如非正常离栏、疑似失联）。
  • 记录处置过程与结果，支持轨迹回溯。

2. 范围边界

  • 本期仅做 Web 端，不包含移动端。
  • 本期不做审核流，仅做预警、处置登记、轨迹查看。
  • 理赔流程不在本期范围。

3. 角色与权限

  • 农场用户：仅查看本农场保后监控数据。
  • 管理员：查看全部农场数据并可统一下发处置要求。

4. 页面结构

  • 筛选区：养殖场、预警类型、处置状态、时间范围。
  • 统计区：监控总数、预警数、处理中、已处置。
  • 列表区：异常预警列表 + 处置操作。
  • 详情弹窗：基本信息、预警信息、轨迹信息、处置记录。

5. 核心字段

字段说明是否必填
farmId养殖场ID是
warningNo预警编号是（唯一）
livestockNo牲畜编号是
collarNo项圈编号是
warningType预警类型（存活异常/离栏异常/轨迹异常）是
warningLevel预警等级（低/中/高）是
warningTime触发时间是
status处置状态（待处置/处理中/已处置）系统维护
locationInfo位置描述/GIS点位否
handleRemark处置说明处置时必填
traceData轨迹数据否

6. 功能清单

  • 监控预警分页查询（多条件筛选）。
  • 异常详情查看（含轨迹信息）。
  • 处置登记（更新状态、记录说明、记录处置人和时间）。
  • 轨迹追踪查看（按时间段回放位置点）。

7. 业务规则

  • 预警记录生成后默认状态为“待处置”。
  • 状态流转：待处置 -> 处理中 -> 已处置。
  • 已处置记录不可删除，仅允许追加备注。
  • 所有处置动作需留痕（操作人、时间、内容）。

8. 接口建议

  • GET /biz/insAfter/page：保后预警分页
  • GET /biz/insAfter/detail：预警详情
  • GET /biz/insAfter/trace：轨迹数据
  • POST /biz/insAfter/handle：处置登记
  • GET /biz/insAfter/stats：统计数据

9. 验收标准

  • 可完成筛选查询、详情查看、处置登记、轨迹查看闭环。
  • 权限隔离有效：农场用户不可查看其他农场数据。
  • 处置过程留痕完整，轨迹数据可按时间段查询。
