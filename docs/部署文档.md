# Gago-Fast 部署文档

## 1. 部署架构

- 后端：`snowy-web-app`，Java 11，默认服务端口 `82`（开发脚本映射为 `8082`）
- 前端：`snowy-web`，Vite 构建产物 `dist/`，Nginx 托管
- 数据库：PostgreSQL（当前项目使用 schema `yzx`）
- 缓存：Redis（登录态、验证码、Sa-Token 依赖）

## 2. 环境要求

- JDK `11`
- Maven `3.8+`
- Node.js `18+`、npm
- PostgreSQL（可连通）
- Redis（可连通）

## 3. 关键配置

后端主配置在：
- `snowy-web-app/src/main/resources/application.properties`
- `snowy-web-app/src/main/resources/application-prod.properties`

核心参数：
- 数据库：`spring.datasource.dynamic.datasource.master.*`
- Redis：`spring.redis.*`
- 服务端口：`server.port`

建议：生产环境不要在 `application-prod.properties` 明文保存密码，改为环境变量或密钥服务注入。

## 5. 生产手工部署（前后端分离）

## 5.1 后端打包与启动

```bash
cd /Users/yanzx/kaohe/gago-fast-1
mvn -s settings.xml -Dmaven.repo.local=.m2/repository -DskipTests package
```

产物：
- `snowy-web-app/target/snowy-web-app-2.0.0.jar`

启动示例：

```bash
java -Xmx2G -Xms1G -jar snowy-web-app/target/snowy-web-app-2.0.0.jar \
  --server.port=82 \
  --spring.datasource.dynamic.datasource.master.driver-class-name=com.p6spy.engine.spy.P6SpyDriver \
  --spring.datasource.dynamic.datasource.master.url='jdbc:p6spy:postgresql://<host>:<port>/<db>?currentSchema=yzx&useUnicode=true&characterEncoding=utf-8&useSSL=false&allowPublicKeyRetrieval=true&nullCatalogMeansCurrent=true&useInformationSchema=true' \
  --spring.datasource.dynamic.datasource.master.username='<user>' \
  --spring.datasource.dynamic.datasource.master.password='<password>' \
  --spring.redis.host='<redis_host>' \
  --spring.redis.port='<redis_port>' \
  --spring.redis.password='<redis_password>'
```

## 5.2 前端打包与Nginx发布

```bash
cd /Users/yanzx/kaohe/gago-fast-1/snowy-web
npm i
npm run build
```

产物：
- `snowy-web/dist/`

Nginx 关键规则（示例来自 `docker/nginx/default.conf`）：
- 静态资源：`root /data/web/build;`
- 前端路由：`try_files $uri /index.html;`
- 后端代理：`/api/ -> http://127.0.0.1:82/`

### 5.3 移动端 H5 单独发布

移动端 H5 使用 `snowy-mobile/h5.Dockerfile`，它的 `/api` 代理目标不要再写死在镜像里，而是通过环境变量 `MOBILE_API_UPSTREAM` 指定。

默认值：
- `gago-fast-server:82`

构建：

```bash
cd /Users/yanzx/kaohe/gago-fast-1/snowy-mobile
docker build -t gago-fast-h5:latest -f h5.Dockerfile .
```

运行示例：

```bash
docker run -d \
  --name gago-fast-h5 \
  -p 8080:80 \
  -e MOBILE_API_UPSTREAM=172.16.0.10:82 \
  gago-fast-h5:latest
```

如果在 Docker Compose / Kubernetes 中部署，建议将 `MOBILE_API_UPSTREAM` 设置为后端 Service 的稳定访问地址，例如：
- `gago-fast-server:82`
- `gago-fast-server.basic-platform.svc.cluster.local:82`

验收：

```bash
docker exec gago-fast-h5 cat /etc/nginx/conf.d/upstream.conf
curl -i http://127.0.0.1:8080/api/auth/b/getPicCaptcha
```

如果当前线上是宿主机目录挂载到 `nginx:alpine` 容器，而不是直接跑自定义镜像，可以使用现成脚本：

```bash
cd /Users/yanzx/kaohe/gago-fast-1
./scripts/deploy_mobile_h5_server.sh
```

脚本默认行为：
- 本地执行 `snowy-mobile/npm run build`
- 打包并上传 `snowy-mobile/dist`
- 远端备份旧目录到 `/opt/gago/mobile-backups`
- 替换 `/opt/gago/mobile/dist`
- 重启 `gago-mobile-h5`
- 校验 `http://127.0.0.1:8081/api/auth/b/getPicCaptcha`

可覆盖的环境变量：
- `SERVER_HOST`
- `SERVER_USER`
- `SERVER_PASSWORD`
- `REMOTE_BASE_DIR`
- `REMOTE_BACKUP_DIR`
- `CONTAINER_NAME`
- `REMOTE_PORT`

## 6. Docker 部署

### 6.1 单容器（Nginx + 后端）

使用项目根目录 `Dockerfile`：

```bash
cd /Users/yanzx/kaohe/gago-fast-1
# 先确保前端 dist、后端 jar 已构建
docker build -t gago-fast:latest -f Dockerfile .
docker run -d --name gago-fast -p 80:80 -p 82:82 gago-fast:latest
```

### 6.2 前后端分容器

- 后端镜像：`server.Dockerfile`
- Web 前端镜像：`web.Dockerfile`
- 移动端 H5 镜像：`snowy-mobile/h5.Dockerfile`

推荐直接用统一脚本按服务构建：

```bash
cd /Users/yanzx/kaohe/gago-fast-1

# 后端
SERVICE=server ./scripts/build_push_aliyun.sh

# Web 管理端
SERVICE=web ./scripts/build_push_aliyun.sh

# 移动端 H5
SERVICE=mobile-h5 ./scripts/build_push_aliyun.sh
```

默认镜像名：
- `SERVICE=server` -> `gago-fast-server`
- `SERVICE=web` -> `gago-fast-web`
- `SERVICE=mobile-h5` -> `gago-fast-mobile-h5`
- `SERVICE=all-in-one` -> `gago-fast`（仅兼容旧链路）

### 6.2.1 本地没有 Docker daemon 时，改用远端 Docker 构建/推送

如果当前电脑无法启动 Docker Desktop / Colima，可以继续在本地构建产物，但把镜像构建和推送交给服务器 `182.92.207.79` 的 Docker 引擎完成：

```bash
cd /Users/yanzx/kaohe/gago-fast-1

SERVICE=server BUILD_MODE=remote ./scripts/build_push_aliyun.sh
SERVICE=web BUILD_MODE=remote ./scripts/build_push_aliyun.sh
SERVICE=mobile-h5 BUILD_MODE=remote ./scripts/build_push_aliyun.sh
```

脚本会按服务自动选择本地构建步骤：
- `server`：只执行 Maven 打包
- `web`：只执行 `snowy-web/npm run build`
- `mobile-h5`：只执行 `snowy-mobile/npm run build`

可覆盖的环境变量：
- `SERVICE`
- `BUILD_MODE`
- `IMAGE_NAME`
- `IMAGE_TAG`
- `IMAGE_PLATFORM`
- `SERVER_HOST`
- `SERVER_USER`
- `SERVER_PASSWORD`
- `REMOTE_BUILD_DIR`

## 9. 常见故障与处理

### 9.1 前端全部 `503`

原因：后端未启动或启动失败。

排查：
- `lsof -iTCP:8082 -sTCP:LISTEN -n -P`
- `tail -n 100 .dev/logs/backend.log`

### 9.2 数据库域名解析失败（`UnknownHostException`）

现象：后端日志出现数据库域名无法解析，进程退出。

处理：
- 检查 DNS / 网络
- 必要时在本机加 hosts 映射后刷新 DNS
- 再执行 `./scripts/dev_up.sh`

### 9.3 Redis超时导致登录/验证码异常

现象：日志出现 `RedisCommandTimeoutException`。

处理：
- 确认 Redis 地址、端口、密码
- 检查网络连通和 Redis 负载
- 重点关注 `spring.redis.*` 配置
